# Методичка — «Робот MeArm» (сборка + управление из pygame)

Аудитория: дети ~13 лет, **уже прошли курс игр на pygame** (цикл, события,
`get_pressed`, списки, экранные кнопки). 11 занятий + вводное. Раздел открывается
по адресу `/robot/` (на главной сайта пока **не** показан — ссылки нет).

## Архитектура
**ПК (Ubuntu, pygame) → USB → Arduino Nano → (I2C) → PCA9685 → 4 сервопривода.**
pygame шлёт строки `"номер,угол\n"` через `pyserial`; прошивка `serial_servos.ino`
парсит и крутит серво через драйвер **PCA9685** (`Adafruit_PWMServoDriver`,
`angleToPulse`: 0–180° → ~150–600). Углы ограничены и в Python, и в Arduino.
**Питание серво — от USB** (через 5 В Nano на вход V+ PCA9685), отдельного блока нет.

## Что купить (электроники в школе нет, комплектов несколько)
**Уже в наборе MeArm (не покупать):** 4 металл-серво, PCB+кабель, винты/гайки/
ножки/глазки/шестигранник. **Тела нет** — режем/печатаем сами по DXF/STL.

**На каждый комплект:** Arduino **Nano** + USB; драйвер **PCA9685**; breadboard;
провода «мама-папа»; крестовая отвёртка; лист 3 мм акрил/фанера или PLA;
*(желательно)* USB-зарядка/хаб на 2 А. **Отдельный блок питания не нужен.**
**Общее:** ПК с Ubuntu (Python + `pygame-ce` + `pyserial` + Arduino IDE с библиотекой
**Adafruit PWM Servo Driver**), мультиметр (опц.), липучка/струбцина для базы.

## Подготовка ПК (один раз)
- `pip install pygame-ce pyserial`
- Arduino IDE, плата «Arduino Nano», порт `/dev/ttyUSB0` (Nano с CH340 — поставить драйвер).
- Библиотека **Adafruit PWM Servo Driver** (Управление библиотеками).
- Доступ к порту: `sudo usermod -aG dialout $USER` → перезайти в систему.

## Схема подключения (урок 6 — главный по железу)
- 4 провода Nano↔PCA9685: `5V→VCC`, `GND→GND`, `A4→SDA`, `A5→SCL`.
- Питание серво: `Nano 5V → PCA9685 V+` (от USB). Земля общая (через GND выше).
- Серво в каналы: `0`=база, `1`=плечо, `2`=локоть, `3`=захват.

## Безопасность (повторять каждое занятие)
1. Питание серво — **от USB через 5 В Nano** на V+ PCA9685. USB ПК даёт ~0,5 А:
   **двигать по одному суставу плавно**; надёжнее — USB-зарядка/хаб 2 А.
2. **Общая земля**: GND Nano и PCA9685 соединены.
3. Если Nano перезагружается/серво дёргаются — нехватка тока: подать внешние 5 В на V+.
4. **Не крутить суставы руками** под питанием — сорвёт шестерёнки.
5. Базу робота **закрепить** к столу.

## Занятия

**Часть 1 — сборка**
- **Урок 1** — покупки и тело: что есть/докупить; режем лазером (DXF) или печатаем
  (STL); снять плёнку с акрила; разложить детали. Ссылка на офиц. фото-гайд MeArm v3.
- **Урок 2** — сервопривод: 3 провода, PWM, 0–180°; **главное правило: центрировать
  в 90° ПЕРЕД установкой качалки**. Заливаем `one_servo_test.ino` (ставит 90°).
- **Урок 3** — основание + база (серво №0): центрировать, закрепить, качалку «вперёд».
- **Урок 4** — плечо (№1) и локоть (№2): центрировать оба; параллельный механизм тяг.
- **Урок 5** — захват (№3): центрировать, надеть «наполовину открытым», проверить ход.

**Часть 2 — код (pygame)**
- **Урок 6 (главный по железу)** — схема (`img/wiring.svg`): 4 провода I2C Nano↔PCA9685
  (`5V→VCC, GND→GND, A4→SDA, A5→SCL`); питание серво от USB на `V+`; серво в каналы 0–3;
  **общая земля**; нюанс тока USB. Заливаем `center_all.ino` — все серво в 90° (проверка).
- **Урок 7** — первый скетч: ставим библиотеку Adafruit PWM Servo Driver; Arduino IDE
  (плата Nano/порт/загрузка), `pwm.setPWM(канал, 0, импульс)`, `setup`/`loop` ↔ игровой цикл.
- **Урок 8** — pygame ↔ Arduino: прошивка `serial_servos.ino` (парсит `"i,angle"`),
  `robot_link.connect()` + `control_one.py` (база со стрелок). Про порт и `dialout`.
  **Режим показа**: без робота код не падает (можно писать заранее).
- **Урок 9** — все 4 сустава: `control_arm.py`, клавиши, **пределы (clamp)**, HUD углов.
- **Урок 10** — пульт + запись поз: `robot_panel.py`, поза = список из 4 углов,
  программа = список поз; кнопки мышкой (как «Играть заново» в «Космосе»).
- **Урок 11** — проект «переставь предмет»: `demo_pick.py`, `PROGRAM` = список поз +
  паузы + захват. Идеи расширений.

## Файлы кода (`robot/code/`)
- `arduino/center_all.ino` — все серво в 90° через PCA9685 (сборка/проверка).
- `arduino/one_servo_test.ino` — один серво (канал 0) в 90° / качание.
- `arduino/serial_servos.ino` — основная прошивка (PCA9685, парсер `"i,angle\n"`, клампы,
  каналы 0–3).
- Всем `.ino` нужна библиотека **Adafruit PWM Servo Driver**.
- `robot_link.py` — `connect()` → `Robot` или `_Offline` (режим показа); `send(j,angle)`.
- `control_one.py` (урок 8), `control_arm.py` (9), `robot_panel.py` (10), `demo_pick.py` (11).

## Типичные проблемы
- **Nano перезагружается / серво дёргаются** → нехватка тока по USB. Двигать по одному
  суставу плавно; включить через USB-зарядку/хаб 2 А; в крайнем случае — внешние 5 В на V+.
- **Серво не двигаются совсем** → нет общей земли (GND Nano↔PCA9685) или не залита
  библиотека Adafruit; проверь подключение SDA/SCL (A4/A5).
- **Сустав упирается в одну сторону** → серво не отцентрировали в 90° перед сборкой.
- **Python «нет доступа к порту»** → `usermod -aG dialout`, перезайти; порт `/dev/ttyUSB0`.
- **`No module named serial`** → `pip install pyserial` (без него — «режим показа»).
- **Скорость не та** → 9600 и в Arduino, и в Python.

## Чего избегаем
Классы (кроме готового `Robot`), сложная кинематика, ШИМ-математика. Всё на углах
0–180, списках и том же игровом цикле pygame, что дети уже знают.
