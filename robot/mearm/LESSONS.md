# Методичка — «Робот MeArm» (сборка + управление из pygame через Firmata)

Аудитория: дети ~13 лет, **уже прошли курс игр на pygame** (цикл, события,
`get_pressed`, списки, экранные кнопки) и желательно вводный курс **«Основы электроники»**
(`/robot/basics/`). 11 занятий + вводное. Раздел открывается по адресу `/robot/mearm/`.

## Архитектура (после ревью: Firmata, без своего протокола)
**ПК (Ubuntu, pygame) → pymata4 → USB → Arduino Nano (Firmata) → 4 сервопривода на ножках.**
В Nano один раз заливается стандартная прошивка **FirmataExpress** (или StandardFirmata).
В Python библиотека **pymata4** управляет ножками, а мы прячем её за объектами-суставами
(`arm.base`, `arm.shoulder`, `arm.elbow`, `arm.grip`, у каждого `.angle`). Серво подключены
**прямо к ножкам Nano** (база=9, плечо=10, локоть=11, захват=6) через Firmata-servo — отдельный
драйвер PCA9685 больше не нужен. **Питание серво — от USB**, общая земля.

Почему так (из ревью): детям не нужно знать «протокол» и писать прошивку. Они опрашивают и
управляют ножками из Python, а сложное спрятано в объекты. Меньше деталей (нет PCA9685) — дешевле
и проще.

## Что купить
**Уже в наборе MeArm:** 4 металл-серво, PCB+кабель, винты/гайки/ножки/глазки/ключ. **Тела нет** —
режем лазером (DXF) или печатаем (STL).
**На каждый комплект:** Arduino **Nano** + USB; breadboard; провода «мама-папа»; крестовая
отвёртка; лист 3 мм акрил/фанера или PLA; *(желательно)* USB-зарядка/хаб 2 А.
**Отдельный драйвер серво и блок питания не нужны.**
**Общее:** ПК с Ubuntu (Python + `pygame-ce` + **`pymata4`** + Arduino IDE с библиотекой
**FirmataExpress**), мультиметр (опц.), струбцина для базы.

## Подготовка ПК (один раз)
- `pip install pygame-ce pymata4`
- Arduino IDE, библиотека **FirmataExpress** (Управление библиотеками), плата «Arduino Nano»,
  порт `/dev/ttyUSB0` (CH340 — поставить драйвер).
- Залить прошивку: **Файл → Примеры → FirmataExpress → FirmataExpress** → Загрузить. Один раз.
- Доступ к порту: `sudo usermod -aG dialout $USER` → перезайти.

## Подключение (урок 6 — главный по железу)
- Сигнал серво на ножки Nano: `база=D9, плечо=D10, локоть=D11, захват=D6` (см. `SERVO_PINS` в `arm.py`).
- Питание серво: красный → 5V, чёрный/коричневый → GND (общая земля). Нехватка тока → внешние 5 В
  на питание серво, GND соединить с Nano.

## Безопасность (повторять каждое занятие)
1. USB ПК даёт ~0,5 А: двигать **по одному суставу** плавно; надёжнее — USB-зарядка/хаб 2 А.
2. **Общая земля**: GND внешнего питания (если есть) и GND Nano соединены.
3. Nano перезагружается/серво дёргаются — нехватка тока: внешние 5 В на питание серво.
4. **Не крутить суставы руками** под питанием — сорвёт шестерёнки.
5. Базу робота **закрепить**.

## Занятия
**Часть 1 — сборка**
- **Урок 1** — покупки и тело: что есть/докупить (Nano, макетка, провода — без драйвера); режем/печатаем.
- **Урок 2** — сервопривод: 3 провода, угол 0–180°; **центрировать в 90° ПЕРЕД установкой качалки**.
  Центровка теперь из Python: `python center.py` (нужна залитая Firmata, см. урок 7).
- **Урок 3** — основание + база; **Урок 4** — плечо и локоть; **Урок 5** — захват. Центрировать перед сборкой.

**Часть 2 — код (pygame)**
- **Урок 6 (главный по железу)** — подключение: серво на ножки Nano (`D9/D10/D11/D6`), питание
  5V/GND, общая земля; нюанс тока USB. Схема `img/wiring.svg`.
- **Урок 7 — Заливаем Firmata**: что такое Firmata и зачем; ставим библиотеку **FirmataExpress**,
  заливаем готовый пример (Arduino НЕ программируем), `pip install pymata4`, проверка `center.py`.
- **Урок 8 — pygame ↔ робот**: командуем **объектами** (`arm.base.angle += 2`), а не ножками;
  `arm.py` (`connect()` → `Arm`/режим показа) + `control_one.py`. Порт и `dialout`.
- **Урок 9** — все 4 сустава: `control_arm.py`, клавиши, пределы хранит объект `Servo`, HUD углов.
- **Урок 10** — пульт + запись поз: `robot_panel.py`, поза = `arm.angles()` (список из 4 углов),
  программа = список поз; кнопки мышкой.
- **Урок 11** — проект «переставь предмет»: `demo_pick.py`, `PROGRAM` = список поз + паузы.

## Файлы кода (`robot/mearm/code/`)
- `arm.py` — `connect()` → `Arm` (через pymata4/Firmata) или режим показа. Объекты `Servo`
  (`.angle` с пределами) и `Arm` (`base/shoulder/elbow/grip`, `set(i,angle)`, `angles()`).
- `center.py` — ставит все серво в 90° (замена старому `center_all.ino`).
- `control_one.py` (8), `control_arm.py` (9), `robot_panel.py` (10), `demo_pick.py` (11).
- `arduino/README.txt` — как залить FirmataExpress (своих .ino больше нет).

## Типичные проблемы
- **Nano перезагружается / серво дёргаются** → нехватка тока по USB. По одному суставу; USB-хаб 2 А;
  внешние 5 В на питание серво (общая земля).
- **Серво не двигаются совсем** → не залита Firmata; не та ножка (сверь `SERVO_PINS`); нет общей земли.
- **Сустав упирается в одну сторону** → серво не отцентрировали в 90° перед сборкой (`center.py`).
- **Python «нет доступа к порту»** → `usermod -aG dialout`, перезайти; порт `/dev/ttyUSB0`.
- **`No module named pymata4`** → `pip install pymata4` (без него — «режим показа»).

## Чего избегаем
Своего «протокола» и прошивки Arduino (заливаем готовую Firmata), сложной кинематики, ШИМ-математики.
Всё — на углах 0–180, объектах-суставах, списках и том же игровом цикле pygame, что дети уже знают.
