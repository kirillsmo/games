pygame ↔ Arduino

Цель урока Связать компьютер и Arduino: pygame шлёт угол по USB, робот двигает сустав.

🧠 Идея

Arduino будет слушать USB-порт и крутить серво по командам вида "номер,угол". pygame на компьютере будет эти команды отправлять, когда мы жмём клавиши. Связь по USB называется serial.

1️⃣ Прошивка Arduino

Залей основную прошивку — она принимает команды и крутит все 4 серво:

⬇️ serial_servos.ino

Внутри она читает строку "0,90" и делает servo[0].write(90). Углы ограничены пределами — лишнее само обрежется.

2️⃣ Код на Python (pygame)

Поставь библиотеку для связи (один раз): pip install pyserial. Затем запусти control_one.py — стрелки крутят базу:

from robot_link import connect
robot = connect()          # сам найдёт Arduino

# ... в игровом цикле, по нажатию стрелок ...
angle = max(0, min(180, angle))
robot.send(0, angle)      # сустав 0 (база) на угол
⬇️ control_one.py robot_link.py (связь)
⚠️ Порт на Ubuntu (Nano) У Nano порт обычно /dev/ttyUSB0 (чип CH340; на старых системах нужен его драйвер). Если Python пишет «нет доступа к порту» — добавь себя в группу: sudo usermod -aG dialout $USER, потом перезайди в систему. Скорость порта (9600) должна совпадать в Arduino и в Python.
💡 Робота ещё нет под рукой? robot_link работает и в «режиме показа»: без Arduino программа просто не шлёт команды и не падает. Можно писать код заранее.
