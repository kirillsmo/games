# Методичка — «Робомашинка» (сборка + кокпит на pygame по Bluetooth)

Аудитория: дети ~13 лет, **уже прошли курс игр на pygame** (цикл, события, `get_pressed`,
списки, экранные кнопки `collidepoint`) и желательно курс **руки-робота MeArm** (serial,
`robot_link.py`, режим показа). 11 уроков + бонус. Раздел открывается по адресу `/robot/car/`
(на главной сайта пока **не** показан — как и MeArm; ссылки нет).

Набор: **Elegoo Smart Robot Car Kit V1** — Arduino UNO + плата-расширение, драйвер моторов
**L298N**, 4 TT-мотора, ультразвук HC-SR04 на серво SG90, датчики линии, **Bluetooth-модуль**,
держатель 2×18650. Отдельно есть **ESP32-CAM** (для бонус-урока 12).

## Архитектура
**Ноутбук (Ubuntu, pygame) ⇄ Bluetooth ⇄ Arduino UNO → L298N → 4 мотора; серво+ультразвук; линия.**
Связь **двусторонняя**: pygame шлёт команды строкой, машина шлёт назад телеметрию.
Главное отличие от MeArm: **у машины своя батарея (18650)** → она беспроводная, а Bluetooth —
это «тот же serial по воздуху», `pyserial` работает с портом `/dev/rfcomm0` так же, как с USB.

## Протокол (одна строка = одна команда/сообщение, 9600 бод)
**Ноутбук → машине (команды):**
| Команда | Действие |
|---|---|
| `F,200` | вперёд, скорость 0..255 |
| `B,200` | назад |
| `L,200` | поворот налево («танком») |
| `R,200` | поворот направо |
| `S` | стоп |
| `SCAN,1` / `SCAN,0` | включить/выключить качание датчика и поток расстояний |

**Машина → ноутбуку (телеметрия):**
| Сообщение | Смысл |
|---|---|
| `D,угол,см` | ультразвук под углом серво (0..180), расстояние в см |
| `T,л,с,п` | датчики линии: слева/середина/справа (1 = видит линию) |

Клампы скорости (0..255) и углов — и в Arduino, и в Python.

## Что купить (почти всё в наборе)
**Уже в наборе (не покупать):** UNO + плата-расширение, L298N, 4 мотора с колёсами, HC-SR04 на
серво, датчики линии, Bluetooth-модуль, корпус, крепёж, провода, держатель батарей.
**Докупить:** 2× **18650** + зарядка (часто не в наборе); USB Bluetooth-донгл, если у ноутбука
нет Bluetooth; *(бонус)* модуль **ESP32-CAM** + USB-переходник для его прошивки.
**Общее:** ПК с Ubuntu (Python + `pygame-ce` + `pyserial` + Arduino IDE), отвёртка, коробок/подставка.

## Подготовка ПК (один раз)
- `pip install pygame-ce pyserial`
- Arduino IDE, плата «Arduino UNO», порт `/dev/ttyACM0` (или `/dev/ttyUSB0` с драйвером CH340).
- Доступ к порту: `sudo usermod -aG dialout $USER` → перезайти в систему.
- Bluetooth: спарить модуль (PIN `1234`/`0000`), привязать `sudo rfcomm bind 0 <MAC>` → `/dev/rfcomm0`.

## Подключение (сверь с реальной платой-расширением!)
- L298N: левые `ENA=5, IN1=7, IN2=8`; правые `ENB=6, IN3=9, IN4=11`.
- Серво (качает датчик): `D3`. Ультразвук: `TRIG=12, ECHO=13`.
- Датчики линии: `A0` (слева), `A1` (середина), `A2` (справа).
- Bluetooth-модуль: его `TX→RX(0)`, `RX→TX(1)` платы (перекрёст). На время заливки — **снять модуль**.

## Безопасность (повторять каждое занятие)
1. **Первый тест моторов — машина на коробке** (колёса в воздухе), иначе уедет со стола.
2. **18650 — строго по полярности**; заряжать со взрослым; не путать +/−.
3. Питание машины **выключать**, когда подключаешь/переставляешь провода.
4. Не гонять машину людям/животным под ноги; не ронять со стола.
5. Bluetooth-модуль занимает `RX/TX` — на время загрузки скетча снимать.

## Занятия

**Часть 1 — сборка и первая жизнь**
- **Урок 1** — знакомство: что в наборе (мозг/мускулы/глаза), архитектура (`img/architecture.svg`),
  отличие от MeArm (своя батарея → беспроводная), безопасность 18650.
- **Урок 2** — сборка шасси: моторы, колёса, батарейный отсек, платы; идея поворота «танком».
  Опора на офиц. гайд Elegoo — урок про понимание, а не каждый винт.
- **Урок 3** — мозг и провода: L298N (`ENA/IN1/IN2`, `ENB/IN3/IN4`), `img/wiring-l298n.svg`;
  заливаем `arduino/drive_test.ino` — «вперёд 2 c». Машина криво → поменять провода мотора.

**Часть 2 — прошивка Arduino**
- **Урок 4** — команды моторам: протокол `"буква,скорость"`, поворот танком, `analogWrite` скорость;
  `arduino/serial_drive.ino`, командуем из Serial Monitor (9600). «Тот же serial, что у MeArm».
- **Урок 5** — Bluetooth: подключаем модуль (перекрёст RX/TX), спаривание, `rfcomm bind`,
  `/dev/rfcomm0`. **Главная мысль: Bluetooth = serial по воздуху**, протокол не меняется.
- **Урок 6 (главный по железу)** — глаза: HC-SR04 (`pulseIn`, `us/58`), серво качает датчик,
  телеметрия `D,угол,см` и `T,л,с,п`. Заливаем основную прошивку `arduino/car_firmware.ino`
  (умеет всё). `SCAN,1` запускает поток расстояний.

**Часть 3 — кокпит на pygame**
- **Урок 7** — `car_link.py` (`connect()` → `Car`/`_Offline`; `send`, **`read`**, `online`) +
  `drive_keys.py` (стрелки, команда только при смене). **Режим показа**: без машины не падает,
  телеметрия понарошку.
- **Урок 8** — кокпит `cockpit.py`: скорость `+`/`-` (по KEYDOWN, кламп 80..255), полоска скорости
  (как полоска жизней в «Космосе»), индикатор связи `car.online`.
- **Урок 9** — радар `radar.py`: читаем `D,угол,см`, `split`, тригонометрия (`cos/sin`,
  `img/radar.svg`), точки по цвету дальности, бегущий луч. Работает и в режиме показа.
- **Урок 10** — автопилот `autopilot.py`: автотормоз (`front_cm < 20` → нельзя вперёд, мигает
  «СТОП»), датчики линии `T,..` тремя кружками. **Решение принимает Python**, не Arduino.
- **Урок 11 (проект)** — миссия `mission.py`: маршрут = **список команд**; «Записать шаг»
  (`route.append`), «Играть» (проход по списку с таймингом, как `robot_panel.py`), «Очистить».
  Та же идея «списка объектов», что в играх. Идеи расширений.

**Бонус**
- **Урок 12 (продвинутый, опц.)** — камера `fpv.py`: ESP32-CAM раздаёт MJPEG по Wi-Fi, ловим кадры
  (`FF D8`..`FF D9`) в фоновом потоке, рисуем `pygame.image.load(BytesIO)`. Нет камеры — заставка
  «нет сигнала», не падает. Прошивка камеры (CameraWebServer) — со взрослым.

## Файлы кода (`robot/car/code/`)
- `car_link.py` — `connect()` → `Car` (serial по Bluetooth/USB) или `_Offline` (режим показа).
  `send(cmd)`, `read()` → строка телеметрии/None, `online`, `close()`. **Двусторонняя версия
  `robot_link.py`** (добавлен `read`).
- `drive_keys.py` (7), `cockpit.py` (8), `radar.py` (9), `autopilot.py` (10), `mission.py` (11),
  `fpv.py` (12, бонус).
- `arduino/drive_test.ino` (3) — едет вперёд 2 c (проверка моторов/L298N).
- `arduino/serial_drive.ino` (4) — езда по командам из Serial (танковый поворот, ШИМ).
- `arduino/car_firmware.ino` (6, основная) — Bluetooth + моторы + серво/ультразвук + линия,
  двусторонний протокол; нужна стандартная библиотека **Servo**.

## Типичные проблемы
- **Нет `/dev/rfcomm0`** → не спарен модуль или не сделан `rfcomm bind`; проверь питание модуля.
- **Машина едет криво / поворот не туда** → поменять местами два провода мотора на L298N
  (или `HIGH`/`LOW`/направление в коде). Это нормальная настройка.
- **Скетч не заливается** → снят ли Bluetooth-модуль с RX/TX на время загрузки?
- **`No module named serial`** → `pip install pyserial` (без него — «режим показа»).
- **Python «нет доступа к порту»** → `usermod -aG dialout`, перезайти.
- **Радар пустой на реальной машине** → не послан `SCAN,1`; проверь серво и питание (серво
  «просаживает» 5 В — при дёрганье добавь питание/конденсатор).
- **Скорость/тарабарщина в порту** → 9600 и в модуле, и в Arduino, и в Python.
- **Машина «дёргается» при езде** → слабые батареи 18650; зарядить/заменить.

## Чего избегаем
Классы (кроме готового `Car`), сложную математику (только `cos/sin` для радара — со связью
с уроком математики). Всё — на командах-строках, списках и том же игровом цикле pygame, что
дети уже знают. Ум машины держим в Python (понятно и легко менять), Arduino — простой исполнитель.
