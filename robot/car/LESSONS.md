# Методичка — «Робомашинка» (сборка + кокпит на pygame через Firmata)

Аудитория: дети ~13 лет, **уже прошли курс игр на pygame** и курс руки **MeArm** (там освоили
Firmata, pymata и объекты), желательно вводный курс **«Основы электроники»** (мотор, H-мост
L298N). 11 уроков + бонус. Раздел открывается по адресу `/robot/car/`.

Набор: **Elegoo Smart Robot Car Kit V1** — Arduino UNO + плата-расширение, драйвер моторов
**L298N**, 4 TT-мотора, ультразвук HC-SR04 на серво SG90, датчики линии, Bluetooth-модуль,
держатель 2×18650. Отдельно — **ESP32-CAM** (для бонус-урока 12).

## Архитектура (после ревью: Firmata, без своего протокола)
**Ноутбук (Ubuntu, pygame) → pymata4 → USB/Bluetooth → Arduino UNO (Firmata) → L298N → 4 мотора;
серво+ультразвук; линия.** В UNO один раз заливается **FirmataExpress**. Из Python мы управляем
ножками через pymata4, спрятав всё за объектами: `car.left`/`car.right` (моторы),
`car.radar` (серво + ультразвук-сонар), `car.line` (3 датчика). Команды: `car.forward()`,
`car.left_turn()`, `car.stop()`, `car.radar.look(a)`, `car.radar.measure()`, `car.line.read()`.

Почему так (из ревью): «крутить колёса И разбирать протокол одновременно» — это не для
начинающих (уровень 1 курса вуза). Дети просто опрашивают/управляют ножками из Python, а сложное
(сонар, ШИМ) спрятано в объекты. Качание радара теперь делает Python (а не прошивка) — «ум на
ноутбуке», понятнее и легче менять.

## Честно про DC-моторы (важная правка по ревью)
- Машину **уводит в сторону**: моторы крутятся по-разному → нужна **калибровка** (`car.left.trim`,
  `car.right.trim` в `robot.py`, урок 8). Даже с ней прямой ход — приблизительный.
- «Память пути» (урок 11) повторяется **похоже, но неточно**: повтор по таймеру + проскальзывание +
  разный заряд батарей. Точный повтор требует **энкодеров с PID** или **шаговых моторов** —
  это продвинутый уровень, о чём прямо сказано в уроках 8 и 11.

## Что купить (почти всё в наборе)
**Уже в наборе:** UNO + плата-расширение, L298N, 4 мотора, HC-SR04 на серво, датчики линии,
Bluetooth-модуль, корпус, крепёж, держатель батарей.
**Докупить:** 2× **18650** + зарядка; USB Bluetooth-донгл (если у ноутбука нет BT); *(бонус)*
**ESP32-CAM** + USB-переходник.
**Общее:** ПК с Ubuntu (Python + `pygame-ce` + **`pymata4`** + Arduino IDE с **FirmataExpress**),
отвёртка, коробок/подставка.

## Подготовка ПК (один раз)
- `pip install pygame-ce pymata4`
- Arduino IDE, библиотека **FirmataExpress** (с зависимостями Servo/Ultrasonic), плата «Arduino UNO»,
  порт `/dev/ttyACM0` (или `/dev/ttyUSB0` с CH340).
- Залить: **Файл → Примеры → FirmataExpress → FirmataExpress** → Загрузить (один раз).
- Доступ к порту: `sudo usermod -aG dialout $USER` → перезайти.
- Bluetooth (опц.): спарить (PIN `1234`/`0000`), `sudo rfcomm bind 0 <MAC>` → в `robot.py`
  указать `com_port="/dev/rfcomm0"`. **Честно:** Firmata по BT подтормаживает — для начала USB.

## Подключение (сверь с реальной платой-расширением!)
- L298N: левые `ENA=5, IN1=7, IN2=8`; правые `ENB=6, IN3=9, IN4=11`.
- Серво радара: `D3`. Ультразвук: `TRIG=12, ECHO=13`.
- Датчики линии: `A0/A1/A2` (на Uno это ножки `14/15/16`).
- Bluetooth-модуль: `TX→RX(0)`, `RX→TX(1)`. На время заливки — снять модуль.
- Все пины заданы вверху `robot.py`.

## Безопасность (повторять каждое занятие)
1. **Первый тест моторов — машина на коробке** (колёса в воздухе).
2. **18650 — строго по полярности**; заряжать со взрослым.
3. Питание машины **выключать**, когда переставляешь провода.
4. Не гонять под ноги людям/животным.
5. Bluetooth-модуль занимает RX/TX — на время загрузки Firmata снимать.

## Занятия
**Часть 1 — сборка и первая жизнь**
- **Урок 1** — знакомство: что в наборе, архитектура, отличие от MeArm (своя батарея), 18650.
- **Урок 2** — сборка шасси: моторы, колёса, батареи, платы; поворот «танком».
- **Урок 3** — мозг и провода: L298N, `arduino/drive_test.ino` — единственный «сырой» скетч,
  проверка моторов **до** Firmata. Криво → поменять провода мотора.

**Часть 2 — Firmata и моторы из Python**
- **Урок 4 — Моторы из Python**: залить **FirmataExpress** (один раз), командовать объектами
  `car.forward()/left_turn()/stop()`; `robot.py`, поворот «танком» внутри объекта.
- **Урок 5 — Bluetooth**: та же Firmata по воздуху; `rfcomm bind`, `com_port="/dev/rfcomm0"`;
  **честный блок про лаг Firmata по BT** (для начала — USB).
- **Урок 6 — Глаза машины**: читаем датчики из Python — `car.radar.look/measure` (сонар Firmata),
  `car.line.read()`. Качание делает Python. Нужна FirmataExpress (сонар).

**Часть 3 — кокпит на pygame**
- **Урок 7** — первый заезд: `drive_keys.py`, объект `car`, стрелки; режим показа без машины.
- **Урок 8** — кокпит `cockpit.py`: скорость `+/-`, полоска, `car.online`; **+ калибровка `trim`
  и честный блок: идеально прямо DC-моторы не едут**.
- **Урок 9** — радар `radar.py`: Python сам качает датчик (`look`/`measure`), тригонометрия,
  точки по дальности. Работает и в режиме показа.
- **Урок 10** — автопилот `autopilot.py`: автотормоз (`car.radar.measure() < 20`), линия
  `car.line.read()` тремя кружками. Решение принимает Python.
- **Урок 11 (проект)** — миссия `mission.py`: маршрут = **список действий**; запись/проигрывание;
  **честный раздел «почему повтор неточный»** (DC-моторы → энкодеры+PID / шаговики).

**Бонус**
- **Урок 12** — камера `fpv.py`: ESP32-CAM раздаёт MJPEG по Wi-Fi (независимо от Firmata).

## Файлы кода (`robot/car/code/`)

## Типичные проблемы
- **`No module named pymata4`** → `pip install pymata4` (без него — «режим показа»).
- **Радар всегда ~200/не мерит** → залита StandardFirmata вместо FirmataExpress (сонар не работает);
  не те пины TRIG/ECHO; нет питания серво.
- **Машина едет криво** → калибровка `trim` (урок 8) или поменять провода мотора на L298N.
- **Тормозит/дёргается по Bluetooth** → особенность Firmata по BT; перейти на USB или réже опрашивать радар.
- **Нет `/dev/rfcomm0`** → не спарен модуль / нет `rfcomm bind`.
- **«Нет доступа к порту»** → `usermod -aG dialout`, перезайти.
- **Машина дёргается при езде** → слабые 18650, зарядить.

## Чего избегаем
Своего «протокола» и прошивки Arduino (заливаем готовую Firmata), одновременного «руления +
парсинга» (это и было главной претензией ревью). Классы пишем не дети — они уже готовы в `robot.py`;
дети пользуются понятными объектами и списками в знакомом игровом цикле pygame.
