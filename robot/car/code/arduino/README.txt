ПРОШИВКА ARDUINO ДЛЯ РОБОМАШИНКИ — теперь это стандартная Firmata.
==================================================================

Раньше под машинку писали свою прошивку с «протоколом команд» (F,200 / SCAN,1 ...).
Больше так не делаем — это сложно для начала (крутить колёса И разбирать протокол
одновременно). Теперь заливаем в плату ОДИН раз готовую прошивку FirmataExpress,
и весь «ум» машины живёт в Python (robot.py и программы уроков).

Остаётся ровно один маленький «сырой» скетч — drive_test.ino — он нужен в уроке 3,
чтобы убедиться, что моторы и драйвер L298N вообще работают, ещё ДО Firmata.

ЧТО СДЕЛАТЬ ОДИН РАЗ (для уроков 4 и дальше)
--------------------------------------------
1. Открой Arduino IDE.
2. «Инструменты → Управление библиотеками…», установи «FirmataExpress» (автор MrYsLab).
   Вместе с ней нужны библиотеки Servo, Ultrasonic — менеджер предложит их поставить.
3. «Файл → Примеры → FirmataExpress → FirmataExpress».
4. Плата «Arduino Uno», порт обычно /dev/ttyACM0 (или /dev/ttyUSB0 с чипом CH340).
5. «Загрузить» (→). Больше плату трогать не нужно.

ПОДКЛЮЧЕНИЕ (сверь с реальной платой-расширением — пины могут отличаться!)
-------------------------------------------------------------------------
   L298N левые:  ENA=5, IN1=7, IN2=8     правые: ENB=6, IN3=9, IN4=11
   Серво радара: D3      Ультразвук HC-SR04: TRIG=12, ECHO=13
   Датчики линии: A0/A1/A2 (на Uno это ножки 14/15/16)
Эти номера заданы в robot.py — поменяй там, если подключил иначе.

ПИТАНИЕ
-------
Моторы питаются от аккумуляторов 18650 (через L298N), НЕ от платы. Логику Arduino
питаем от USB. Земля общая. Первый тест — машина на коробке (колёса в воздухе)!

БЕСПРОВОДНОЙ ВАРИАНТ (Bluetooth)
-------------------------------
Firmata умеет работать и по Bluetooth: в robot.py укажи com_port="/dev/rfcomm0".
Но честно: по Bluetooth Firmata «болтливее» и может подтормаживать, особенно при
постоянном опросе радара. Для первых заездов надёжнее USB-кабель; беспроводной режим —
как улучшение, когда всё уже работает.
