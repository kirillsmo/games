# План уроков — «Змейка» на Python

Аудитория: **возраст ~13 лет**, знают переменные, `if`, `for`, списки.
Формат: **7 уроков** по ~45 минут. Каждый урок заканчивается **работающей программой**.
Всего: ~5–6 уроков чистого кода + 1 урок на доработку и расширения.

## Подготовка (делаем один раз со всем классом)

```
cd snake
python -m venv .venv
source .venv/bin/activate     # Windows:  .venv\Scripts\activate
pip install pygame-ce
```

Каждый урок — это один файл в папке `games/snake/code/`. Дети открывают его, читают,
запускают, потом копируют в следующий файл и добавляют новый кусочек. **Не**
заставляйте их начинать с пустого файла — у них всегда должна быть работающая
программа, которую они расширяют.

Запуск файла:

```
python games/snake/code/step1_window.py
```

## Урок 1 — Окно, которое открывается и закрывается

**Файл:** `games/snake/code/step1_window.py`
**Что видят дети:** тёмное окно. Закрывается крестиком.
**Новые понятия:** `import`, цвета RGB как кортежи, `while True`, цикл событий, FPS.

**Сначала объясните главную модель** — любая игра каждый кадр делает одно и то же:

1. Читает ввод (события)
2. Обновляет мир
3. Рисует мир
4. Ждёт следующий кадр

Покажите пальцем каждый блок в файле и назовите его.

**Частые ошибки:**
- Забыли `pygame.display.flip()` → окно остаётся серым.
- Забыли цикл событий → окно зависает, macOS пишет «программа не отвечает».

**Мини-задание:** поменяйте `BLACK` на другой RGB. Сделайте окно 800×600.

## Урок 2 — Рисуем один квадрат на сетке

**Файл:** `games/snake/code/step2_square.py`
**Что видят дети:** один зелёный квадрат стоит на колонке 5, строке 10.
**Новые понятия:** координаты в клетках против пикселей, `pygame.Rect`, `pygame.draw.rect`.

**Объясните:** змейка живёт на сетке (30 × 20 клеток). Экран — в пикселях. Мы
умножаем координаты клетки на `CELL`, чтобы получить пиксели. Этот один приём
делает всё остальное в игре простым.

Нарисуйте на доске:

```
колонка 0  1  2  3  4  5
строка 0 [ ][ ][ ][ ][ ][ ]
строка 1 [ ][ ][ ][ ][ ][ ]
строка 2 [ ][ ][ ][ ][ ][ ]   <- (колонка 3, строка 2) значит x=3, y=2
```

**Мини-задание:** переместите квадрат в `(0, 0)`. Потом в правый нижний угол.
Какие `x` и `y` его туда поставят?

## Урок 3 — Змейка — это список

**Файл:** `games/snake/code/step3_snake.py`
**Что видят дети:** три зелёных квадрата в ряд. Первый ярче — это голова.
**Новые понятия:** список кортежей, `snake[0]` = голова, `snake[1:]` = хвост, `for` по списку.

**Объясните:** змейка — это просто **список позиций `(x, y)`**. Индекс 0 — это
голова. Всё, что мы будем делать дальше — двигать, растить, проверять
столкновения — это просто операции со списком.

**Мини-задание:** добавьте 4-й и 5-й сегмент тела. Сделайте, чтобы змейка
начинала вертикально, а не горизонтально.

## Урок 4 — Змейка двигается сама

**Файл:** `games/snake/code/step4_move.py`
**Что видят дети:** змейка скользит вправо и уходит за край экрана.
**Новые понятия:** `direction` как кортеж `(dx, dy)`, `insert(0, ...)` + `pop()`, `clock.tick(10)`.

**Это самый важный урок.** Приём движения такой:

> Добавляем новую голову спереди. Убираем последний кусочек хвоста. Змейка не
> «двигалась» — она вырастила новую голову и потеряла старый хвост.

Нарисуйте это на доске стикерами — детям очень нравится.

```
Было:    [Г][.][.][.]
Шаг 1:   [Н][Г][.][.][.]   ← вставили новую голову
Шаг 2:   [Н][Г][.][.]      ← убрали хвост
```

**Частые ошибки:**
- Забыли `clock.tick(10)` → змейка летит через экран на 60 кадрах в секунду.
- Перепутали `pop()` (конец) и `pop(0)` (начало).

**Мини-задание:** сделайте, чтобы змейка шла вверх. Сделайте по диагонали (`direction = (1, 1)`).

## Урок 5 — Стрелки

**Файл:** `games/snake/code/step5_keys.py`
**Что видят дети:** настоящая игра-змейка (пока без еды). Стрелки её поворачивают.
**Новые понятия:** `pygame.KEYDOWN`, `if / elif`, защита от разворота на 180°.

**Объясните защиту от 180° явно.** Почему перед `direction = (0, -1)` стоит
проверка `direction != (0, 1)`? Потому что если змейка едет вниз, а ребёнок
нажал вверх, новая голова окажется *внутри тела* — мгновенная смерть. Защита
запрещает разворот.

**Частые ошибки:**
- Цикл событий стоит вне `while True` → клавиши не работают (это был исходный баг!).
- Забыли `elif` и пишут только `if` — несколько клавиш могут сработать на одно
  событие (редко, но стоит упомянуть).

**Мини-задание:** добавьте WASD как альтернативное управление.

## Урок 6 — Еда и рост

**Файл:** `games/snake/code/step6_food.py`
**Что видят дети:** красное яблоко. Змейка растёт на 1, когда ест.
**Новые понятия:** определение функции (`def`), `random.randint`, условный `pop()`, f-строки для заголовка.

**Объясните приём роста:**

> Обычный шаг: вставили голову, убрали хвост (длина не меняется).
> Съели яблоко: вставили голову, **не** убрали хвост (длина +1).

То есть «съесть» — это буквально «пропустить один `pop()`». Вот и всё.

Ещё объясните, почему `spawn_food` — это цикл `while True`: мы не хотим, чтобы
яблоко появилось *внутри* змейки.

**Мини-задание:** сделайте яблоко за 5 очков вместо 1. Добавьте на поле второе
яблоко.

## Урок 7 — Проигрыш и рестарт

**Файл:** `games/snake/code/step7_gameover.py`
**Что видят дети:** удар об стену или о себя заканчивает игру. Пробел — заново.
**Новые понятия:** булев флаг (`game_over`), оператор `in` для списков, вывод текста, упаковка состояния в функцию (`new_game`).

**Объясните:** «Как мы узнаём, что игра окончена?» → два условия:

1. `new_head` вышла за пределы `0..COLS` или `0..ROWS` → удар об стену.
2. `new_head in snake` → ударились о себя.

Оператор `in` делает всю проверку «есть ли эта клетка где-то в моём теле» в одну
строку. Отличный момент, чтобы похвалить Python.

**Мини-задание:** покажите счёт на экране вместо заголовка. Сделайте, чтобы
змейка ускорялась каждые 5 яблок (`clock.tick(10 + score // 5)`).

## Урок 8 — Похвастаться и расширить (по желанию)

Дети выбирают **одно** расширение и показывают его на следующем занятии. Примеры:

| Расширение | Какой навык тренирует |
|---|---|
| Рекорд, сохранённый в файл | работа с файлами |
| Стены с переходом на другую сторону | остаток от деления (`%`) |
| Два игрока (WASD + стрелки) | параллельное состояние |
| Звук при поедании | `pygame.mixer` |
| Препятствия-блоки на поле | ещё один список, тот же приём столкновений |
| Стартовый экран «Нажми пробел» | новое состояние игры |

Финальный образец — `snake.py` в корне проекта (то же самое, что урок 7, просто
немного убрано).

## Советы по преподаванию

- **Всегда запускайте после каждого изменения.** Если ребёнок не видит эффект —
  он теряет нить. Когда что-то сломалось, `print` — нормальный помощник.
- **Имена важнее, чем хитрость.** `direction`, `food`, `snake`, `head`, `tail` —
  дети должны читать код вслух как английский (или русский) текст.
- **Пока избегайте:** классов, словарей, `enumerate`, генераторов списков,
  лямбд, `*args`. Всё это понятно само по себе, но каждое — отдельный урок.
- **Когда ребёнок застрял:** попросите прочитать цикл вслух, по одной строчке.
  90% багов всплывают при чтении.
- **Классический баг**, который они поймают: если очень быстро нажать две
  клавиши (например вверх, потом влево, когда едешь вниз), змейка всё равно может
  погибнуть. Защита от 180° проверяет *уже применённое* направление, а не
  следующее в очереди. Это чинится, но добавляет сложности — оставьте как
  «задание со звёздочкой».
