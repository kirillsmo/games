# Как выложить сайт с играми на GitHub Pages (со своим доменом)

Здесь — пошаговая инструкция, как сделать так, чтобы дети открывали курс по
адресу вроде `games.школа.рф`. Хостинг бесплатный, HTTPS бесплатный.

> Что именно публикуем: **курсы-уроки** (HTML-страницы). Сами игры (`*.py`) —
> это десктопные программы на Python, они запускаются на компьютере командой
> `python games/snake/code/snake.py`, а не в браузере. На сайте — уроки с кодом
> и объяснениями.

---

## Структура сайта (рассчитана на много игр)

```
index.html            ← главная-каталог: список всех игр
shared.css            ← общий стиль для всех курсов
games/
  snake/              ← всё про одну игру в одной папке
    index.html        ← старт курса
    urok1..7.html     ← уроки
    code/             ← python-файлы (запускаются на компьютере)
    LESSONS.md        ← гид для учителя
DEPLOY.md  README.md
.nojekyll  .gitignore
```

Добавить новую игру = создать папку `games/имя/` и добавить карточку на главной.
Подробно — в `README.md`, раздел «Как добавить новую игру».

- `.nojekyll` — говорит GitHub Pages «не обрабатывай сайт, просто отдавай файлы».
- `.gitignore` — не даёт залить лишнее (`.venv/`, кэш Python).

---

## Шаг 0. Что нужно один раз

- Аккаунт на [github.com](https://github.com) (бесплатный).
- Установленный `git` (проверь командой `git --version`).

---

## Шаг 1. Создать репозиторий на GitHub

1. Зайди на github.com → кнопка **New** (или https://github.com/new).
2. Имя репозитория, например `games-course`.
3. Тип **Public** (для бесплатного Pages нужен публичный).
4. **Не** ставь галочки «Add README / .gitignore» — у нас уже всё есть.
5. Нажми **Create repository**. GitHub покажет адрес вида
   `https://github.com/ТВОЙ_АККАУНТ/games-course.git` — он понадобится дальше.

---

## Шаг 2. Залить проект

Открой терминал в папке проекта (`snake`) и выполни по строчкам.
Если git в этой папке ещё не настроен:

```
git init
git add .
git commit -m "Курс Змейка на Python"
git branch -M main
```

Подключи свой репозиторий (подставь свой адрес из шага 1) и отправь:

```
git remote add origin https://github.com/ТВОЙ_АККАУНТ/games-course.git
git push -u origin main
```

> Git может попросить логин. Вместо пароля GitHub нужен **токен**:
> github.com → Settings → Developer settings → Personal access tokens →
> Tokens (classic) → Generate new token, права `repo`. Вставь токен вместо пароля.

---

## Шаг 3. Включить GitHub Pages

1. В репозитории на сайте: **Settings → Pages** (меню слева).
2. В разделе **Build and deployment**:
   - Source: **Deploy from a branch**
   - Branch: **main**, папка **/ (root)** → **Save**.
3. Через 1–2 минуты вверху появится адрес вида
   `https://ТВОЙ_АККАУНТ.github.io/games-course/`.
4. Открой его — должен открыться курс. На этом этапе уже всё работает (без
   своего домена).

---

## Шаг 4. Подключить свой домен

### Вариант А (рекомендую) — поддомен, например `games.школа.рф`

1. В **настройках DNS** у того, кто управляет доменом школы, добавь запись:

   ```
   Тип:   CNAME
   Имя:   games               (часть слева от домена)
   Цель:  ТВОЙ_АККАУНТ.github.io.
   ```

2. На GitHub: **Settings → Pages → Custom domain** — впиши `games.школа.рф`,
   нажми **Save**. (GitHub сам создаст в репозитории файл `CNAME` — не удаляй его.)

### Вариант Б — корневой домен, например `школа.рф`

Вместо CNAME добавь четыре A-записи на IP GitHub:

```
Тип A,  Имя @ →  185.199.108.153
Тип A,  Имя @ →  185.199.109.153
Тип A,  Имя @ →  185.199.110.153
Тип A,  Имя @ →  185.199.111.153
```

Затем так же впиши домен в **Settings → Pages → Custom domain**.

> ⚠️ Корневой домен школы скорее всего уже занят основным сайтом.
> Поэтому почти всегда правильнее **поддомен** (вариант А) — он не трогает
> существующий сайт.

---

## Шаг 5. Включить HTTPS

Когда DNS «доедет» (от нескольких минут до пары часов), на
**Settings → Pages** станет активной галочка **Enforce HTTPS** — поставь её.
Теперь сайт открывается по `https://` с замочком.

---

## Как обновлять курс потом

Поменял страницу — три команды, и сайт обновится сам за минуту:

```
git add .
git commit -m "Поправил урок 4"
git push
```

---

## Если что-то не открывается

- **Страница 404 сразу после push** — подожди 1–2 минуты, Pages собирается не мгновенно.
- **Открывается код вместо сайта** — проверь, что в Settings → Pages выбрана папка
  **/ (root)**, а не `/docs`.
- **Свой домен не работает** — DNS обновляется не сразу; проверь запись командой
  `nslookup games.школа.рф`. Должно указывать на `github.io`.
- **Нет галочки Enforce HTTPS** — значит сертификат ещё выпускается, зайди позже.
