# Контрольные вопросы (самопроверка) на страницах уроков

Блоки «🧠 Проверь себя» — это самотест прямо на странице урока. Работает на GitHub Pages
(чистая статика, без сервера) и даже офлайн.

## Почему ответы нельзя угадать или подсмотреть
- Ответ вводится **свободным текстом** — нет кнопок-вариантов, перебирать нечего.
- В HTML лежит **не ответ, а его хэш** (SHA-256, «посоленный» `data-id` и прогнанный 2000 раз).
  Из хэша ответ не восстановить, во `View Source` его не видно.
- Правильный ответ нужно **найти в уроке** — на это и расчёт: подумать → посмотреть → ответить.
- Объяснение («почему так») открывается **только после верного ответа**.
- Прогресс запоминается в браузере (localStorage), решённые вопросы остаются зелёными.

> Честная оговорка: очень короткий числовой ответ (например «да/нет» или 1–2 цифры) теоретически
> можно подобрать скриптом, перебрав хэши. Поэтому формулируй ответы как **слова, термины,
> имена методов или осмысленные числа** — тогда перебор бессмыслен. Не делай вопросов с ответом
> «да/нет».

## Как добавить блок на страницу урока

1. Перед `<div class="pager">` вставь блок:

```html
<div class="quiz" data-iter="2000">
  <h2>🧠 Проверь себя</h2>
  <div class="quiz-q" data-id="car4-method"
       data-hash="ХЭШ1 ХЭШ2"
       data-look="Подсказка после 3 ошибок: где смотреть.">
    <p class="quiz-prompt">Каким методом объекта car машина едет вперёд?</p>
    <p class="quiz-hint">Одно английское слово.</p>
    <div class="quiz-explain">✅ Верно — forward: car.forward(180).</div>
  </div>
</div>
```

2. Перед `</body>` подключи скрипт (путь зависит от глубины страницы; уроки лежат на 2 уровня
   ниже корня):

```html
<script src="../../quiz.js"></script>
```

3. Поля блока:
   - `data-id` — уникальный идентификатор вопроса (он же «соль»). Не меняй потом — иначе хэш
     перестанет совпадать.
   - `data-hash` — один или несколько допустимых хэшей через пробел (синонимы ответа).
   - `data-look` — (необязательно) подсказка, которая появится после 3 неверных попыток.
   - `.quiz-hint` — мягкая подсказка, видна сразу.
   - `.quiz-explain` — объяснение, показывается только после верного ответа.

## Как посчитать хэши ответов

```bash
python3 tools/quizhash.py <data-id> "ответ1" "ответ2" ...
```

Например:

```bash
python3 tools/quizhash.py car4-method "forward" "car.forward" "forward()"
```

Скрипт печатает готовую строку `data-hash="…"` — вставь её в блок. Несколько ответов = несколько
синонимов, любой из них засчитается.

## Нормализация ответа (что прощается ученику)
Перед сравнением и ввод, и эталон приводятся к одному виду (в `quiz.js` и `quizhash.py` — одинаково):
нижний регистр, `ё`→`е`, запятая→точка (для дробей: `1,5`→`1.5`), лишние символы убираются,
пробелы схлопываются, крайние точки/дефисы срезаются. То есть `Car.Forward()`, `forward`,
`FORWARD` дадут один результат. Параметр `--iter` (число повторов хэша) должен совпадать с
`data-iter` у блока (по умолчанию 2000).
