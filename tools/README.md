# Контрольные вопросы (самопроверка) на страницах уроков

Блоки «🧠 Проверь себя» — самотест прямо на странице урока. Работает на GitHub Pages
(чистая статика, без сервера) и офлайн.

## Как устроено и почему нельзя «протыкать» наугад
- Ответ выбирается **кнопкой** — печатать ничего не нужно (удобно без русской раскладки).
- Варианты при каждой загрузке **перемешиваются** — нельзя запомнить «правильная — вторая».
- После **неверного** ответа вопрос блокируется на **30 секунд «на почитать»** (таймер от
  подбора): за это время надо вернуться в урок и найти ответ. Перебор становится бессмысленным.
- В HTML лежит **не «какой вариант верный», а хэш** правильного ответа (SHA-256, посоленный
  `data-id`, прогнанный 2000 раз) — в исходнике страницы не видно, какой вариант засчитается.
- Объяснение («почему так») открывается **только после верного ответа**.
- Прогресс хранится в браузере (localStorage). Внизу блока — кнопка **«↺ Сбросить ответы»**
  (очищает прогресс по всем урокам).

## Как добавить блок на страницу урока

1. Перед `<div class="pager">` вставь блок:

```html
<div class="quiz" data-iter="2000" data-cooldown="30">
  <h2>🧠 Проверь себя</h2>
  <p style="color:var(--muted);margin-top:-2px">Выбери ответ. Ошибёшься — 30 секунд на перечитать.</p>

  <div class="quiz-q" data-id="car4-method" data-hash="ХЭШ_ПРАВИЛЬНОГО">
    <p class="quiz-prompt">Каким методом объект car едет вперёд?</p>
    <div class="quiz-options">
      <button type="button" class="quiz-opt">forward</button>   <!-- правильный -->
      <button type="button" class="quiz-opt">go</button>
      <button type="button" class="quiz-opt">drive</button>
      <button type="button" class="quiz-opt">move</button>
    </div>
    <div class="quiz-explain">✅ Верно — forward: car.forward(180).</div>
  </div>
</div>
```

2. Перед `</body>` подключи скрипт (уроки лежат на 2 уровня ниже корня):

```html
<script src="../../quiz.js"></script>
```

3. Поля:
   - `data-iter` — число повторов хэша (по умолчанию 2000). Должно совпадать с `--iter` в генераторе.
   - `data-cooldown` — пауза в секундах после неверного ответа (по умолчанию 30).
   - `data-id` — уникальный идентификатор вопроса (он же «соль»). Не меняй потом.
   - `data-hash` — хэш(и) **правильного** варианта. Можно несколько через пробел, если у верного
     ответа есть синонимичные написания.
   - `.quiz-opt` — варианты-кнопки (один правильный + несколько неверных). Текст правильной кнопки
     должен совпадать с тем, из чего считался `data-hash` (с точностью до нормализации).
   - `.quiz-explain` — объяснение, показывается только после верного ответа.

## Как посчитать хэш правильного ответа

```bash
python3 tools/quizhash.py <data-id> "<текст правильной кнопки>"
```

Например:

```bash
python3 tools/quizhash.py car4-method "forward"
```

Скрипт печатает готовую строку `data-hash="…"`. Неверные варианты (distractors) в генератор
вписывать не нужно — для них хэши не считаются.

## Нормализация (что считается «тем же ответом»)
И текст кнопки, и эталон приводятся к одному виду (в `quiz.js` и `quizhash.py` — одинаково):
нижний регистр, `ё`→`е`, запятая→точка, лишние символы убираются, пробелы схлопываются,
крайние точки/дефисы срезаются. Поэтому `Firmata`, `firmata`, `FIRMATA` — одно и то же, а
`300 Ом` нормализуется в `300 ом`.

## Массовая генерация

Блоки во все уроки игр и робототехники вставлены генератором
`tools/build_game_quizzes.py`: в нём лежат все вопросы (по 2 на урок), он считает хэши,
перемешивает варианты и идемпотентно вставляет блок перед `.pager` и подключает `quiz.js`
(повторный запуск пропускает уроки, где блок уже есть). Чтобы добавить вопрос — впиши его в
структуру `Q` и запусти `python3 tools/build_game_quizzes.py`.

## Проверка перед публикацией
Убедись, что у каждого вопроса ровно **один** вариант совпадает с `data-hash` (правильный),
а остальные — нет. Удобно прогнать быстрым скриптом на Node, который хэширует каждую кнопку
тем же `quizhash` (см. историю — так проверялись демо-вопросы).
